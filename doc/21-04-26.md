# Drag UX fix (text highlight issue)

Goal: Prevent text selection while dragging cards

Why:
Dragging causes browser text highlight → looks buggy

Approach:
- Disable selection only during drag
- Restore after drag ends

Done when:
- No blue highlight appears while dragging
- Text can still be selected when NOT dragging

# Event conflict (dblclick vs mousedown)
Goal: Separate drag and edit interactions

Problem:
mousedown (drag) conflicts with dblclick (edit)

Approach:
- Add movement threshold OR delay
- If cursor moves → treat as drag
- If quick double click without movement → edit

Done when:
- Drag feels smooth
- Double click reliably enters edit mode
- No accidental triggers

# State → DBMS migration
Goal: Persist app state in database instead of useState

Scope:
- Store card position
- Store card content
- Sync on change

Approach:
- Define schema (id, position, text)
- Replace local state updates with DB writes
- Load initial state from DB

Done when:
- Refresh doesn’t lose data
- UI reflects DB state correctly
